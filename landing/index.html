Why Toolgate?

Every MCP billing SDK (Cloudflare paidTool, MCPay, Nevermined) treats payment as a binary gate: paid or blocked. That’s wrong for real products. Toolgate gives developers programmable control over what happens at the billing decision point — degrade gracefully, allow a trial call, estimate cost first, or block with a helpful 402. Plus fiat-first Stripe payments with prepaid credits, so no per-call payment handshake.

5 lines to add billing

my-mcp-server.ts 
import { ToolGate, createMcpAdapter } from "@tkorkmaz/toolgate";

const gate = new ToolGate({ publisherKey: "tg_pub_xxx" });
const mcp = createMcpAdapter(gate);

mcp.paidTool("premium_search", {
  price: 0.05,                    // $0.05 per call
  description: "AI-powered deep search",
  inputSchema: {
    type: "object",
    properties: { query: { type: "string" } },
    required: ["query"],
  },
  handler: async ({ query }) => {
    return { results: await deepSearch(query) };
  },
  fallback: async ({ query }) => {
    return { results: await basicSearch(query), note: "Top up for premium" };
  },
});

mcp.registerAll(server);

How it works

Toolgate sits inside your MCP server. When an agent calls a paid tool, the SDK checks the caller's prepaid credit balance, executes the handler, and returns a receipt.

1

Install & wrap

npm install @tkorkmaz/toolgate and wrap your tool handlers with paidTool(). Set a price — static, dynamic, or tiered.

2

Callers top up once

When balance is low, the SDK returns a Stripe Checkout link. Users top up with a credit card — then all subsequent calls are instant. No per-call payment friction.

3

You get paid

Every paid call generates a receipt. Revenue flows to your Stripe account via Connect. Track everything with built-in metering and hooks.

What no other MCP billing SDK does

Every alternative offers a binary gate: paid or blocked. Toolgate gives you programmable billing decisions and graceful execution control.

Graceful Fallback

When a caller can't pay, return a degraded response instead of a hard 402 block. Every other SDK hard-blocks. Toolgate converts.

onPaymentFailed: "fallback",
fallback: async (input) => basicResult(input)

Execution Policy new

Programmable billing decisions per call. Execute, fallback, allow once, or return a cost estimate — based on balance, tier, usage, or input.

policy: { decide: async ({ balance, tier }) =>
  balance >= 0.50 ? "execute" : "fallback" }

Prepaid Credits

Users top up once via Stripe Checkout. Subsequent calls deduct from balance instantly — no per-call 402 handshake. Others do per-call crypto negotiation.

// User tops up $5 → gets 100 calls at $0.05 each
// No payment friction on each call

Dynamic Pricing

Price based on input complexity, output size, or any custom logic. Others only support static per-tool pricing.

price: (input) => input.text.length * 0.001

Postpaid Metering

Charge after execution based on actual duration, tokens, or any metric. No other MCP billing SDK does this.

price: "postpaid",
meter: (i, o, m) => ({ amount: m.durationMs * 0.0001 })

Automatic Refund on Error

If your handler throws, the caller's balance is automatically restored. Never charge for broken tools.

// Handler throws → balance refunded automatically
// No manual refund logic needed
